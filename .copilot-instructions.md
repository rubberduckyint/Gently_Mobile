# Copilot Instructions for Gently Project

## Project Overview

This is a full-stack TypeScript monorepo for the Gently app, featuring:

- **Backend**: Next.js API routes with tRPC, Better-Auth authentication
- **Mobile**: Expo React Native app
- **Database**: PostgreSQL with Drizzle ORM
- **Authentication**: Better-Auth with Google OAuth and magic links
- **Email**: Gmail SMTP for transactional emails

## Code Style & Patterns

### General Guidelines

- Use TypeScript with strict typing
- Prefer functional components and hooks
- Use proper error handling with try/catch blocks
- Follow the existing file structure and naming conventions

### Authentication

- Use Better-Auth client for all authentication operations

### File Structure

- `apps/mobile/` - Expo React Native mobile app
- `apps/nextjs/` - Next.js web application
- `packages/` - Shared packages (api, auth, db, email, validators)
- Environment files: root `.env` for server, `apps/mobile/.env` for mobile

### API & Database

- Use tRPC for type-safe API calls
- Drizzle ORM for database operations
- API routes follow `/api/auth/*` pattern for authentication

### Mobile Development

- Use Expo Router for navigation
- Environment variables prefixed with `EXPO_PUBLIC_`
- Google Sign-In configured with proper URL schemes

### Dependencies

- Package manager: pnpm
- Monorepo tool: Turbo
- Mobile: Expo, React Native
- Backend: Next.js, tRPC, Better-Auth
- Database: PostgreSQL, Drizzle ORM

## Common Tasks

- Start development: `pnpm dev` (runs all apps)
- Mobile development: Use Expo Go or development build
- Database migrations: Use Drizzle Kit
- Environment setup: Ensure all `.env` files are properly configured
